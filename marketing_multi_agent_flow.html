Multi-Agent Marketing Optimization System

This document describes how the OpenAI-based program in marketing_multi_agent_system.py ingests CSV budget and performance feeds, runs three channel specialist agents plus a supervisor orchestrator (and an optional reflector), and produces weekly spend recommendations that exhaust a fixed annual budget evenly across the year (total annual sum divided by 52 weeks).

1. Business problem in one paragraph

Marketing runs paid Google Ads, LinkedIn campaigns, and email programs. Each week, new performance rows arrive as CSV exports. Finance sets an annual budget (your numbers live in data/budget_allocation.csv). The system must recommend how much to spend this week on each channel such that the three amounts add up exactly to the weekly envelope, while using recent performance to shift spend toward stronger channels and away from weaker ones—without exceeding the annual plan when repeated week over week.

2. Data inputs (four CSV files)

2.1 Budget allocation (data/budget_allocation.csv)

One row per channel. Column annual_budget_usd is the annual dollars assigned to that channel. The program sums all rows to get the total annual budget, then computes weekly_total = sum / 52. Edit the CSV to match your plan (for example, three rows that sum to 100,000 for a $100k year).

2.2 Google performance (data/google_performance.csv)

Weekly rows with typical paid search / PMax style metrics: impressions, clicks, CTR, cost, conversions, conversion value, CPA, impression share, quality score.

2.3 LinkedIn performance (data/linkedin_performance.csv)

Weekly rows aligned to LinkedIn Campaign Manager style reporting: impressions, clicks, CTR, spend, leads, CPL, CPC, engagement rate, video views.

2.4 Email performance (data/email_performance.csv)

Weekly rows aligned to ESP reporting: sends, delivered, opens, click rates, unsubscribes, attributed conversions and revenue.

3. Agent roles

  • Google specialist agent — Reads only the Google CSV (recent tail). Emits JSON: summary, signals, health score, and a suggested weight delta versus baseline.
  • LinkedIn specialist agent — Same pattern for LinkedIn metrics.
  • Email specialist agent — Same pattern for email metrics.
  • Supervisor orchestrator agent — Consumes all three specialist JSON outputs plus the budget table. Performs an explicit PARA cycle (Planning, Reasoning, Acting, Reflecting, Adaptation) and outputs recommended USD for each channel for the current week.
  • Reflector agent (optional) — Second pass that critiques the supervisor output and proposes one alternative allocation that still sums to the weekly total—useful for adaptation and human review.

4. PARA cycle (what the supervisor is asked to do)

  1. Planning — Clarify this week’s goals (efficiency, scale, testing, etc.).
  2. Reasoning — Merge the three specialists’ scores and narratives.
  3. Acting — Choose concrete weekly spends for Google, LinkedIn, and Email that sum to the required weekly total.
  4. Reflecting — Surface risks, thin data, and assumptions.
  5. Adaptation — State what to watch next week and how policy would change if signals move.

After the model responds, the Python layer normalizes the three dollar amounts so floating-point drift never breaks the constraint google + linkedin + email = weekly_total.

6. How to run

pip install -r requirements.txt
set OPENAI_API_KEY=your_key_here
python marketing_multi_agent_system.py --data-dir ./data --json-out weekly_allocation.json

Optional: OPENAI_MODEL in the environment overrides the default model. Use --skip-reflection to omit the reflector agent for faster runs.

7. Operational notes

  • Weekly envelope — The program assumes a steady weekly burn rate of annual_budget/52. If your finance team uses a custom fiscal calendar, adjust the divisor in code or preprocess the budget CSV.
  • Row window — Only the most recent rows of each performance file are sent to the model (configurable via max_rows in code) to keep prompts small.
  • Ground truth — LLM recommendations are not a substitute for platform-specific constraints (minimum spends, pacing, account caps). Treat output as a decision-support starting point.
